# audit:a11y

TRIGGER when: audit, check, review, or scan for accessibility — "is this a11y-clean", "any a11y issues", "run a11y on X". Auto-eligible after `/ui:component:compose` creates a new component.

Compare frontend source against the a11y baseline. Report deviations as `file:line` entries by severity. CLEAN runs emit no table.

## Arguments

$ARGUMENTS

- Path or component name → audit that scope.
- `--changed` → audit only files in `git diff --name-only` (staged + unstaged).
- `--top N` → in sweep mode, show the top N worst offenders (default 10).
- No arguments → see §2.

---

## 1. Manifest

Read `./manifest.json`. If missing, halt with a pointer to `/repo:manifest`.

Filter `packages` to `isFrontend: true` and `framework` in (`react`, `next`). If none qualify, halt — no frontend to audit.

Per qualifying package, capture:

| Field | Use |
|---|---|
| `framework` | gates Next-specific checks (5.9) |
| `componentsDir` | default scope when no arg, no diff |
| `primitivesDir` | scanned for dialog/overlay primitives (5.4) |
| `hooksDir` | scanned for focus-trap and reduced-motion helpers (5.4, 5.7) |
| `tokensDir` | used for color-contrast checks if it contains color tokens |

Also read top-level `conventions.principles` — drives severity bumps in §6.

---

## 2. Resolve scope

Priority order:

1. Explicit path or component name → audit that.
2. `--changed` → `git diff --name-only HEAD` plus unstaged. Keep `.tsx` / `.jsx` files under a frontend package's `componentsDir`.
3. No argument → try `--changed` first; if empty, fall back to every file under each frontend `componentsDir`.

If a name resolves to multiple matches, list them and ask which.

Record the mode: `single` / `changed` / `sweep`.

---

## 3. Parse targets

Parse `.tsx` / `.jsx` with a TypeScript/Babel JSX AST. Regex only for quick scans.

Per file capture: every element node with tag, attributes (static + dynamic), children outline, source line. Capture spread props (`{...rest}`); elements where required ARIA might come in via spread become **warnings**, not blockers.

---

## 4. Discover color tokens (optional)

If `tokensDir` is set, scan it. Recognized shapes:

- TS/JS file exporting an object whose leaves are CSS color strings (hex / rgb / hsl).
- CSS file declaring `--color-foo: #aabbcc;`.
- JSON tokens file (W3C design-tokens or Style Dictionary) with `"$value"` color leaves.

Build `{ tokenName → resolvedColor }`. Resolve aliases. Skip color-contrast checks for any unresolved token. If no tokens or no color tokens, skip the contrast check entirely — never guess from hardcoded class strings.

---

## 5. Checks

Each check defines one baseline. A check that holds emits nothing. A nit earns its row only when it would survive a second reader's review.

### 5.1. Semantic HTML

- `<div>` / `<span>` with `role` set to `"button"`, `"link"`, `"checkbox"`, `"radio"`, `"switch"`, `"tab"`, `"option"`, `"menuitem"`.
- `<div>` used as clickable (`onClick` / `@click`, no `role`).
- Page-level component (default export named `Page` / `Layout`, or under routes/pages dir) with no `<main>` landmark → **nit**.

### 5.2. ARIA correctness

- Invalid `aria-*` names (typos: `aria-lable`, `aria-decribed-by`).
- Redundant ARIA on native elements (`<button role="button">`, `<a role="link">`, `<nav role="navigation">`).
- Missing required attributes for the role: `role="checkbox"` without `aria-checked`; `role="combobox"` without `aria-expanded` + `aria-controls`; `role="tabpanel"` without `aria-labelledby`; `role="dialog"` without `aria-label` or `aria-labelledby`; `role="img"` without `aria-label` or `aria-labelledby`.
- Boolean ARIA set to non-boolean strings (`aria-hidden="yes"`).

### 5.3. Keyboard navigation

- Element with `onClick` but no keyboard handler when not natively focusable (`<div onClick>` without `tabIndex` and key handler).
- Positive `tabIndex` (`tabIndex={1}` or higher).
- Custom interactive widget lacking expected key handlers for its role (`role="listbox"` without arrow keys, `role="dialog"` without Escape). If the project ships a roving-focus hook, check the hook is invoked instead of inline handlers.

### 5.4. Focus management

- `outline: none` / `outline-none` / `focus:outline-none` without a compensating indicator (`focus:ring-*`, `focus-visible:*`, or equivalent).
- Dialog/overlay components (`role="dialog"` or a `primitivesDir` primitive matching `/dialog|overlay|modal/i`) not referencing a `hooksDir` focus-trap helper. If no focus-trap helper exists, downgrade to warning.
- Overlays missing focus restoration on close (no `onCloseAutoFocus`, no equivalent ref restoration).

### 5.5. Label association

Flag `<input>`, `<textarea>`, `<select>` without an accessible name:

- No `aria-label`.
- No `aria-labelledby`.
- No associated `<label htmlFor>` or wrapping `<label>`.
- No `id` referenced by a sibling label.

Also flag icon-only buttons (`<button>` whose only child is `<svg>` or icon component) without `aria-label`.

### 5.6. Color contrast (only if tokens discovered)

For each element with foreground + background token (scan `className` / `class` for token names from §4's map):

- Resolve both to RGB.
- Compute WCAG contrast ratio.
- Flag pairs below **4.5:1** normal text, **3:1** large text (≥18pt or ≥14pt bold), **3:1** non-text UI / graphical objects.

Ambiguous token pairing → downgrade to warning.

### 5.7. Reduced motion

For animated elements (CSS animations, transition-bearing classes, `motion.*` from Motion/Framer Motion, React-Transition-Group `<Transition>`), flag any animation without a `prefers-reduced-motion` guard in the file or in an imported motion utility. Guards:

- CSS `@media (prefers-reduced-motion: reduce)` block.
- Framework hook (`useReducedMotion()`).
- Library-level setting in a project motion config.

If a central motion config already disables animations under `prefers-reduced-motion`, skip this check.

### 5.8. Image / media alt text

- `<img>` without `alt`. `alt=""` (decorative) is allowed; missing entirely is a finding.
- `<video>` / `<audio>` without `<track kind="captions">` or `aria-label`.
- SVG icons used as standalone images without `aria-label` or `role="img"`.

### 5.9. Next-specific (only when `framework` is `next`)

- Missing `next/image` `alt`.
- Missing `next/font` fallback fonts (CLS / readability when fonts fail to load).
- `<a>` tags that should be `next/link` when navigation matters (pure anchors fine).

---

## 6. Severity

| Severity | Default checks |
|---|---|
| **blocker** | missing label on form control; missing `alt` on `<img>`; role-required attribute missing; positive `tabIndex`; click handler on non-focusable element with no keyboard handler |
| **warning** | redundant ARIA; invalid `aria-*` name; focus indicator removed without replacement; dialog missing focus trap; reduced-motion guard absent |
| **nit** | semantic-HTML alternatives (`<div role="button">` → `<button>`) when runtime behavior is otherwise compliant; missing landmarks on page-level component |

Bump severity one notch when `conventions.principles` enforces the rule (e.g. mentions "every interactive element must be keyboard-operable").

---

## 7. Output

Lead with the verdict.

| Verdict | When |
|---|---|
| **CLEAN** | no deviations recorded — entire report; stop |
| **DEVIATIONS PRESENT** | only warnings/nits |
| **FAIL** | any blocker |

If `--changed` was used and the diff is empty, say so and exit.

Score for ranking deviated components: `(blockers × 5) + (warnings × 2) + (nits × 1)`.

Branch on mode:

- **`single`** — one section, full findings table.
- **`changed`** — one section per changed component, sorted by score descending. Cap at top **10** unless `--top N`.
- **`sweep`** — lead with a Deviation summary table (top 10 by score; baseline-holders omitted), then full per-component sections for the top-N. List the rest as: `Remaining components with deviations: N (M deviations total). Run /audit:a11y <component> to inspect.`

Deviation summary table:

```
| Rank | Component | Location | Blockers | Warnings | Nits | Score | Headline finding |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | Widget | packages/design-system/src/components/widget/widget.tsx | 2 | 1 | 0 | 12 | label-association: icon-only button missing aria-label |
```

Per-component section:

```
### Widget (packages/design-system/src/components/widget/widget.tsx)

| Severity | Rule | Location | Suggested fix |
| --- | --- | --- | --- |
| blocker | label-association | widget.tsx:42 | wrap <input> with <label> or add aria-label |
| warning | focus-indicator | widget.tsx:18 | replace `outline-none` with `focus-visible:ring-2 focus-visible:ring-accent-500` |
| nit | semantic-html | widget.tsx:7 | render <button> instead of `<div role="button">` |
```

Roll-up: `N components audited · K outside baseline · M deviations (B blockers · W warnings · n nits)`.

## Worked examples (fabricated)

### Missing accessible name

Before:

```tsx
<button onClick={onClose}>
  <CloseIcon />
</button>
```

Finding: `blocker · label-association · widget.tsx:42 · add aria-label="Close" to the <button>`

After:

```tsx
<button onClick={onClose} aria-label="Close">
  <CloseIcon />
</button>
```

### Missing focus trap in a dialog

Before:

```tsx
export function ConfirmDialog({ open, onClose, children }: Props) {
  if (!open) return null

  return (
    <div role="dialog" aria-labelledby="confirm-title">
      <h2 id="confirm-title">Confirm</h2>
      {children}
      <button onClick={onClose}>Cancel</button>
    </div>
  )
}
```

Finding: `warning · focus-trap · confirm-dialog.tsx:5 · wrap the dialog body with the project's focus-trap helper so Tab cycles inside`

After (assuming a `useFocusTrap` hook):

```tsx
export function ConfirmDialog({ open, onClose, children }: Props) {
  const ref = useRef<HTMLDivElement>(null)

  useFocusTrap(ref, open)

  if (!open) return null

  return (
    <div ref={ref} role="dialog" aria-labelledby="confirm-title">...</div>
  )
}
```

---

## Rules

- Read source; never edit.
- The verdict is the deliverable; deviations are evidence. CLEAN is a successful run.
- Don't manufacture nits to justify a non-empty report.
- Suggest fixes using project components and helpers — pull names from the manifest and quick scans of `primitivesDir` / `hooksDir`.
- When a check needs browser-only evidence (computed styles, actual focus order, screen-reader output), state the limitation rather than guess.
- If the project ships `eslint-plugin-jsx-a11y`, note the overlap in the closing summary.
- In `sweep` mode: rank, then truncate.
