# Documentation Audit — Grid module

**Date:** 2026-06-29 · **Scope:** the **Grid module** (`packages/ui/src/modules/grid/`) — its public surface, every internal sub-component, hook, and pure helper, the inline comments throughout, the `ROADMAP.md`, and the module's entries in the curated surface indices. This is a **single-module** sweep, narrower than the repo-wide [2026-06-22](2026-06-22-DOC-AUDIT.md) pass; it re-verifies the grid surface against the heavy churn on the `claude/grid-docs-audit` line since then (content-aware auto-sizing, per-row inline editing, column pinning, CSV export, the `stickyHeader`→`header` config, the accessibility-announcements work). **Method:** the TSDoc standard ([CONVENTIONS.md](../../../../CONVENTIONS.md) §12.1) was the bar; the documentation MCP knowledge base was consulted for the TSDoc tag reference (the live `tsdoc.org` fetch is proxy-blocked, so §12.1 and the already-audited public surface served as the authoritative local standard); the 49-file sweep was delegated one subsystem per subagent ([CLAUDE.md](../../../../CLAUDE.md) §3.3) — structural sub-components, feature sub-components, hooks, and pure helpers + ROADMAP — each returning verified findings. **All edits are comment-only — no code, types, imports, JSX, or formatting changed**; a diff scan confirms both changed hunks sit inside `/** … */` blocks. Verified: `biome check` on the changed files (no fixes) and `turbo run check-types --filter=ui` (tsc across the package and its scripts/docs tsconfigs) green.

This is a living record per [`CONVENTIONS.md`](../../../../CONVENTIONS.md) §12.3, accompanying the curated surface docs in [`../`](..).

## Executive summary

The Grid module is in excellent documentation health. Its public surface — all **34 barrel-exported symbols** across `context`, `grid`, `grid-column-manager`, `grid-editing-types`, `grid-data-types`, and `types` — carries summary-first TSDoc in the house voice: first-sentence summaries, `@param`/`@returns` where signatures aren't self-evident, `@defaultValue` on every defaulted optional field, `@remarks` for caveats, and `{@link}` cross-links throughout. The internal surface holds the same bar: every barrel-invisible exported symbol that carries a doccomment also carries `@internal`, and the prose tracks the code — including the parts most prone to drift after a refactor.

The sweep surfaced **two** comment-only defects, both fixed:

1. **`use-grid-reorder.ts` — stale `@returns` (HIGH).** The doccomment named the returned `items`; the hook returns `itemIds` (from `useSortableList`) and also returns an undocumented `activeId`. Corrected to name `itemIds`, `dndContextProps`, and `activeId`.
2. **`use-grid-column-visibility.ts` — missing option-type summary (LOW).** `GridColumnVisibilityOptions` lacked the one-line `@internal` summary every sibling option type in the module carries (`GridReorderOptions`, `GridColumnAutoSizeOptions`, …). Added `/** Options for {@link useGridColumnVisibility}. @internal */`.

No NORMALIZATION findings: no doccomment uses the non-standard `@default`, every `@defaultValue` is correct, and the lone "Defaults to…" prose (`use-grid-column-visibility.ts:13`) describes uncontrolled-binding fallback behavior, not a literal field default, so it is correctly left as prose. No QUALITY findings: doccomments are summary-first and avoid restating types. The curated indices are current — [`MODULES.md`](../MODULES.md) lists `grid` and [`REFERENCE.md`](../../REFERENCE.md) §1 maps to it (the hub names categories, not individual modules, so no per-symbol entry is owed there).

## What was verified

The value of a single-module sweep is in checking the load-bearing claims against the implementation rather than trusting the prose. These were verified and **hold** (recorded so a future pass need not re-derive them):

- **Public surface, 34 symbols.** Every barrel export is documented; `GridProps<T> = GridDataProps<T>` exposes the public name while `GridDataProps` itself is `@internal`. Defaulted fields (`sortable`, `resizable`, `truncate`, `navigable`, `reorder`, `exportable`, header `position`, the export/column-manager `enabled`/`toolbarButton`/`label`, …) all carry an accurate `@defaultValue`.
- **`ROADMAP.md` "Shipped" claims.** CSV export "scoped to the selection when active" (against `grid-data.tsx` selection filtering and `types.ts`), one-way row-selection mirroring via `enableRowSelection` (`use-grid-table.ts`), multi-sort on `getSortedRowModel` + `enableMultiSort` (`grid-table-options.ts`), faceted `select` filtering (`grid-table-views.ts`), the three separate auto-sizing units, the runtime pin-override map, and `aria-rowindex`/`aria-colindex` under pagination (`grid-row.tsx`, `grid-head.tsx`) — all match code. No backlog item is secretly already shipped.
- **High-drift internal claims.** The `cell`-only render path (no TanStack default value renderer), the `GridReorderableCell` "no longer registers a sortable of its own" rewrite (no `useSortable` present), the `useGridTruncation` single shared `ResizeObserver` / `enabled`-gated mounted-span behavior, the `{@link GridProps.maxHeight}` link target resolving through the type alias, and every grid constant's value-vs-prose (`DEFAULT_ROW_HEIGHT = 44`, `NAV_PAGE_STEP = 10`, `COLUMN_RESIZE_STEP = 16`, …).
- **`@internal` discipline.** Confirmed against `index.ts` (which re-exports nothing from the internal files): every documented hook, sub-component props alias, and pure helper — including export-but-not-surfaced symbols like `GridData`, `GridEditInputs`, `smartSortingFn`, and `rowSortKey` — carries `@internal`.
