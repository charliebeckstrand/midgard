# Documentation Audit — `packages/ui`

**Date:** 2026-06-14 · **Scope:** the **whole package, comments included** — a deeper sweep than the [2026-06-13 pass](2026-06-13-DOC-AUDIT.md), which closed the public-surface gaps. This pass re-verifies that surface against drift since then, then goes into the layers that one deferred: non-exported internal hooks and helpers, inline `//` and `{/* */}` comments, the recipe bodies (kiso token headers, kata module headers), the docs-site/api-reference engine, and the demos. **Method:** the standard (TSDoc *tags* / *syntax*, JSDoc *reference*) was pulled from the documentation MCP knowledge base; the sweep was delegated one subsystem per subagent. **All edits are comment-only — no code, types, imports, JSX, or formatting changed.** Verified: `biome check` (1,532 files, no fixes) and `tsc --noEmit` across all three configs (tsc / scripts / docs) green.

This is a living record per [`CONVENTIONS.md`](../../../../CONVENTIONS.md) §12.3, accompanying the curated surface docs in [`../`](..).

---

## Executive summary

The 2026-06-13 audit left the public surface at a high bar; this sweep confirms it held through the ~110 source files that changed since (the variant-based recipe refactor, the opt-in extended palette, the api-reference engine work, `SidebarItem`/`NavItem`, `MapShipmentChat` → `useActionState`). No surface symbol regressed. The yield this time was concentrated where the prior pass deliberately stopped:

1. **One systemic public-surface gap closed.** 44 `*Variants` types are re-exported through component barrels but defined in `recipes/kata/*.ts`; they carried no TSDoc because they live in the recipe layer the prior pass deferred. Each now opens with a summary-first block naming its component. (The 4 `*Variants` that no barrel re-exports were left — not public.) `JsonValueType`, the one other cross-layer barrel re-export, was likewise documented.
2. **Internal layers documented.** The pdf-viewer internal family (sub-components, hooks, result/option types, `downloadPdf`/`printPdf`), the toast internal hooks (`useToastQueue`, `useToastTimer`), `useScrollLock`'s lock helpers, the date-input/date-picker private utilities, and assorted `assignRef`-style helpers gained answer-first `@internal` summaries with `@returns`/`@remarks`.
3. **Stale comments corrected.** A handful of comments had drifted from the code (enumerated below) — the highest-value find of a "comments included" sweep.
4. **Standard normalization, finished.** Residual non-standard `@default` → `@defaultValue`, prose "Defaults to X" → `@defaultValue`, and a few `{@link}` / `@returns` gaps in the hooks and primitives families.

No public-surface symbol is left without a summary; the internal layers are now documented wherever a reader would otherwise have to infer intent from the body.

## Stale or inaccurate comments fixed

These are the comment-vs-code contradictions the sweep caught — recorded because they are the part most likely to mislead:

- **`menu/menu-item.tsx`** — summary and the inline `MenuItemProps` comment claimed a disabled item renders as a `<span>`. True only for the `href` (link) branch; the plain-button branch keeps a `<button>` with `aria-disabled`. Rewritten to distinguish the two.
- **`menu/menu.tsx`** — the no-role comment stated a bare fact about `role="application"` suppressing AT browse-mode as if it were applied; clarified that it is the rejected alternative and why it is omitted.
- **`recipes/kata/badge.ts`** — the wide-palette comment listed "rose / violet / sky", omitting `mist`. The authoritative extended set is `mist / rose / violet / sky`. Corrected.
- **`recipes/kata/icon.ts`** — the `shaku.icon` slot-projection comment listed "Button, Badge, and Sidebar" but the kiso source (`kiso/shaku/icon.ts`) and the kata that read it include **Nav**. Added Nav.
- **`recipes/kiso/README.md`** §3 — the `kokkaku/` row's Files column listed 11 of the 20 skeleton-form units. Reconciled to the full set.
- **`docs/components/api-reference/doc-description.tsx`** — called the reference chip a "rose chip" twice, but `LinkBadge` renders `<Badge variant="soft">` with the recipe's default `zinc`. Corrected to "soft chip".
- **`components/scroll-area`** — a stale `{/* … */}` JSX comment said cells are focusable "when scrollable / omitted on non-scrolling viewports"; reworded to the actual per-axis behavior.

## Surfaced issues (code-level — left for follow-up)

Per [`CLAUDE.md`](../../../../CLAUDE.md) §1.2, flagged not fixed (a comment-only sweep can't touch code):

- **`pdf-viewer` "Fit to page" is mislabeled.** The button (and its tooltip) calls `fit()`, which sets zoom to `1` (100%) — not a fit-to-viewport scale, though `fitScale` already exists in `usePdfViewerPageScale`. The control is effectively "reset zoom". Needs a code/label decision.
- **`date-picker/date-picker-range-reducer.ts`** — the `pinEndpoint` and `hover` cases are byte-identical (both write `hoverDate`; `pinEndpoint` is the commit-time freeze). Intentional, not a bug, but a refactor hazard; documented in place rather than collapsed.
- **`primitives/locale/context.ts`** — the doccomment on the `[LocaleContext, useLocale]` tuple reads as the hook's doc but sits on the context export. Harmless (`createContext` returns both); noted for tidiness.

## Deferred (out of scope this pass)

- **4 non-barrel `*Variants`** — `LinkVariants`, `RangeSliderVariants`, `ColorPanelVariants`, `ProgressBarFillVariants`: defined in kata but not re-exported by any component barrel, so not on the public surface (§12 documents barrel re-exports).
- **Uniform kata per-file headers** — only ~6 of ~90 kata carry a module header today; adding them to all is a layer-wide convention change, not a stale-comment fix, and would need an agreed grammar (`Layer: kata · …`?) per [`CLAUDE.md`](../../../../CLAUDE.md) §3.1. Recommend a dedicated pass if wanted.
- **Per-token recipe TSDoc** — kiso bodies already carry `Layer:` headers and rich READMEs; per-token comments were not pursued (would be noise on self-evident scale/colour values).
- **Tests & benchmarks** — `__tests__/` and `__benchmarks__/` are governed by the test conventions ([CONVENTIONS.md](../../../../CONVENTIONS.md) §10) and the vitest audit tooling, not this documentation sweep.
