# UI Grid Performance Audit вЂ” Open Findings

Performance audit of `src/modules/grid` (2026-06-28) вЂ” React 19, TanStack Table
v8 + Virtual v3, dnd-kit. The render architecture is sound: the row boundary is
memoized (`GridRow = memo(GridRowImpl)`), cursor and active-cell state ride
external stores that bypass reconciliation, and engine view objects hold stable
identity. Real cost clusters in two places вЂ” synchronous O(NВІ) work on editable
bulk-commit paths, and unbounded per-keystroke/per-mount DOM and filter work;
everything else is bounded O(rows)/O(columns) waste on human-cadence
interactions. Each finding was adversarially verified against the live source
(and `table-core@8.21.3` / dnd-kit internals where relevant): of 140 candidates,
75 were confirmed вЂ” consolidated into the 30 distinct findings below, no
Critical surviving вЂ” and 65 were rejected as already mitigated by existing
memoization/virtualization or miscadenced (per-commit costs mislabelled
per-keystroke). Severity is priority order; line numbers are as of the audit
commit and may drift.

## High

- [ ] **`src/modules/grid/use-grid-editable-mutations.ts:92-96`** (with `use-grid-editable-history.ts:58,64,77`) вЂ” editable bulk commit is O(NВІ). `applyCellWrite` fans a single write across all rows by full-array scan, and the resulting K-change batch (Kв‰€N on select-all) flows into `history.emit`, whose `priorValue` does a `rows.findIndex(getKeyвЂ¦)` plus `editableCols.find` *per change*. Select-all + Enter on 10k rows is ~10вЃё comparisons in one blocking handler; matrix paste (`use-grid-editable-wrapper.ts:344-372`) and fill feed the same path. **Fix:** build a `Map<rowKey,index>` + `Map<columnId,col>` once per `emit` (O(N+C)), then resolve each change by O(1) lookup, and iterate the selection Set directly in `applyCellWrite` в†’ O(N+K). The existing `rowIndexMap` is keyed by row object, not rowKey, so thread a rowKey-keyed map in.
- [ ] **`src/modules/grid/use-grid-table.ts:386-389`** вЂ” `renderRows` re-`.map()`s the full page/filtered set every render whenever pagination or a client transform (the default for client sort) is active, discarding TanStack's reference-stable `getRowModel().rows`. The fresh array busts `rowKeys` (`grid-data.tsx:705`) and `rowIndexMap` (`grid-data.tsx:716`) вЂ” two O(rows) rebuilds вЂ” on every GridData render, including each `columnResizeMode:'onChange'` drag frame and search keystroke. **Fix:** `useMemo` keyed on `table.getRowModel().rows`, which is stable until sort/filter/pagination changes.
- [ ] **`src/modules/grid/use-grid-editable-rows.ts:33-41`** (with `use-grid-editable-augmented-columns.tsx:140`) вЂ” the editable `rowIndexMap` rebuilds an O(N) `Map<T,number>` keyed on `[rows]`; the grid is controlled, so each commit returns a fresh `rows` array, rebuilding every augmented column's `cell`/`cellProps` closures в†’ new `columnDefs` в†’ new `getVisibleCells()` в†’ every windowed row re-renders per edit. **Fix:** mirror the read-only nav path (`use-grid-navigation-columns.tsx:109,133,145`) вЂ” read `rowIndexMapRef.current` inside the closures and keep only the stable ref in deps.
- [ ] **`src/modules/grid/grid-filter.tsx:24`** вЂ” global search forwards every keystroke straight to `table.setGlobalFilter` (`use-grid-table.ts:426`) with no debounce, and client filtering is the default. Each character invalidates `getFilteredRowModel` (full `includesString` over rows Г— value-bearing columns + faceted recompute) and rebuilds the `renderRows`/`rowKeys`/`rowIndexMap` cascade вЂ” O(rowsГ—cols) per keypress. **Fix:** hold the value in local state for caret echo and push to `setGlobalFilter` behind `useDeferredValue` (or a ~150вЂ“250ms debounce); optionally wrap the apply in `startTransition`.
- [ ] **`src/modules/grid/use-grid-truncation.ts:76,87-89`** вЂ” `useLayoutEffect(measure)` has no dep array (re-measures after every commit) and installs one `ResizeObserver` per cell; truncation defaults on, mounted per data cell and header title. Each row scrolled into the virtual window mounts C observers + C layout-reading `measure()` calls. *(Calibration: the costly `Range` + double `getBoundingClientRect` path is gated behind an integer `scrollWidth > clientWidth` fast-path, and the "every resize frame" claim is false вЂ” memoized rows don't re-render during resize.)* **Fix:** gate the layout effect on a content dependency (cheapest correct mitigation), then collapse the per-cell observers into one shared table-level observer.
- [ ] **`src/modules/grid/grid-row.tsx:345-355,395`** вЂ” every non-pinned body cell wraps in `GridReorderableCell` в†’ `useSortable` whenever reorder is enabled (not just mid-drag), and each body cell shares its header's sortable id (`String(col.id)`) вЂ” duplicate ids dnd-kit cannot track cleanly. With no `measuring` strategy on the DndContext (`grid-data.tsx:110`), `closestCenter` runs collision math against every registered droppable per pointer-move; non-virtualized grids make the droppable count unbounded in rows. **Fix:** make only headers droppable; drive the body column-shift from the header's drag state via a CSS variable keyed by column; set `measuring={{ droppable: { strategy: MeasuringStrategy.WhileDragging } }}`; never reuse one sortable id across nodes.

## Medium

- [ ] **`src/modules/grid/grid-row.tsx:323`** вЂ” `GridDataCell` is a plain function while `GridRow`/`GridReorderableCell` are `memo`. A row re-render not reflected in cell props (a per-row `selected`/`loading` toggle) re-runs `col.cellProps?.(row)`, `flexRender`, and `resolveCellTooltip` for all C cells. **Fix:** `React.memo(GridDataCell)` вЂ” cell/col identity is stable across such renders, so unchanged cells bail. Bounded by columns-per-toggle, not the rowsГ—columns the raw finding implied.
- [ ] **`src/modules/grid/use-grid-selection.ts:59-61`** вЂ” `allSelected`/`someSelected` are computed unmemoized in the hook body, re-scanning all `rowKeys` on every GridData render (keystrokes, resize frames, pin/pagination changes); `someSelected`'s `.some()` does a full scan when nothing is selected (the common state). **Fix:** `useMemo(вЂ¦, [rowKeys, selection])`, optionally guarded by `selection.size`.
- [ ] **`src/modules/grid/use-grid-truncation.ts:87-89`** вЂ” distinct from the per-commit measure: the simultaneous observer count is window-rows Г— truncated-columns (hundreds on a wide grid), recreated as rows cross the virtualization boundary. *(The "reflow storm on every scroll across every cell" framing is overstated вЂ” persistent cells keep their observers, and the browser batches RO callbacks into one pre-paint pass.)* **Fix:** one shared/per-table observer, rAF-throttled.
- [ ] **`src/modules/grid/grid-sorting-utilities.ts:45`** (with `grid-table-options.ts:124`) вЂ” `compareSmart`, the default comparator, runs three `.replace()` passes + a `DECIMAL.test` per operand of every comparison; a client sort re-parses each value ~O(log N) times with no per-value key cache. Discrete per-sort-click cost, but a large client-sorted column visibly stalls. **Fix:** decorate-sort-undecorate вЂ” one O(N) pass to `{num|null, str}` keys, then compare keys.
- [ ] **`src/modules/grid/use-grid-column-fit.ts:100-104`** вЂ” the column-fit `ResizeObserver` calls `applyFit` в†’ `fitSizes` (O(columns)) + `table.setColumnSizing` (fresh object, no equality bail) on every callback; `setColumnSizing` is GridData state, so each tick re-renders head + body + footer and re-runs the truncation sweep. *(RO already delivers ~once per frame, so rAF coalescing is largely redundant for the single observed element.)* **Fix:** bail when the computed sizing equals the current `columnSizing`.
- [ ] **`src/modules/grid/grid-head.tsx:85-99`, `src/modules/grid/grid-row.tsx:244`** вЂ” no column/horizontal virtualization; the full column set renders and the header never windows. With hundreds of columns, every windowed row mounts C truncation observers + C `flexRender` + C pinned lookups, scaling with column count. Known backlog item (`ROADMAP.md:88`); extreme-width only. **Fix:** a second `@tanstack/react-virtual` axis over `visibleColumns` (windowing header + body, keeping pinned columns), or a documented column-count ceiling.
- [ ] **`src/modules/grid/grid-data.tsx:456`** (with `recipes/kata/table.ts:86`) вЂ” non-resizable grids render `table-layout: auto` (only the resizable branch adds `table-fixed`), so the browser measures intrinsic content width of every cell in every mounted row on first paint and on content change. *(`resizable` defaults true, so this needs explicit `resizable={false}` plus a large non-virtualized set.)* **Fix:** add `table-fixed` + a `getSize`-driven colgroup to the non-resizable path, or expose an opt-in `layout='fixed'`.
- [ ] **`src/modules/grid/use-grid-editable-augmented-columns.tsx:120-137`** (with `grid-editable-context.ts:70-97`) вЂ” the augmented `cell(row)` runs `formatCell(row,col)` per windowed cell on each commit-driven body re-render with no cache; separately the cell-slice selector allocates a `cellKey` string unconditionally before the `extraCells.has` check. **Fix:** cache `formatCell` by `(rowKey, columnId, value)` or format inside the memoized `GridEditableCell`; guard the `cellKey` alloc with `extraCells.size > 0`.
- [ ] **`src/modules/grid/use-grid-navigation-columns.tsx:43-55`** вЂ” the active-cell layout effect does a `closest()` walk, toggles `data-active` (matching a `ring-2` selector в†’ style recalc), and calls `scrollIntoView` (forced layout) on every cursor move, even when the cell is already in view; held-arrow autorepeat makes it a per-frame forced-reflow loop (old + new active cell each move). **Fix:** gate `scrollIntoView` on the cell actually being outside the scroll viewport (rect compare or `IntersectionObserver`).
- [ ] **`src/modules/grid/grid-data.tsx:766-789`** вЂ” the `GridContext` value embeds the volatile `selection` Set, but its sole in-module consumer `GridHeaderCell` (`grid-head.tsx:162`, non-memoized) reads only `allSelected`/`someSelected`/`toggleAll`/sort/pin. Each toggle changes the Set identity в†’ context identity в†’ re-executes all C header-cell bodies (memoized children bail, so no paint amplification вЂ” modest waste). **Fix:** drop `selection` from the head-facing context; the Set already reaches the body by prop.

## Low

- [ ] **`src/modules/grid/grid-head.tsx:151-163`** вЂ” `GridHeaderCell` not memoized; all C router bodies re-run on every selection/sort/pin change before the memoized children bail. Cheap per unit; follows from the Medium-tier context split.
- [ ] **`src/modules/grid/grid-column-filter-button.tsx:75-82`** вЂ” recomputes `uniqueValues` + `JSON.stringify`s facets on every header render even when the sheet is closed; bites a high-cardinality select column being resize-dragged. **Fix:** gate on `open`; key the memo off the faceted-Map reference, not a serialized array.
- [ ] **`src/modules/grid/grid-editable.tsx:227`** вЂ” a fresh `selection` config object literal is passed to `GridData`; harmless today (GridData isn't memoized) but forecloses wrapping `GridData` in `memo`. **Fix:** hoist into a `useMemo`.
- [ ] **`src/modules/grid/grid-data.tsx:448-458`** вЂ” `resolveResizeLayout` rebuilds the `<colgroup>` with live `getSize()` per column every render; verified cheap (arithmetic per column) and listed only for completeness вЂ” the claimed compounding with truncation re-measure does not occur.
- [ ] **`src/modules/grid/use-grid-editable-mutations.ts:20-46`** вЂ” `applyBulkFill`/`collectFillCoords` materialize the full rectangle as Coords + a dedupe Set up front; linear and one-shot, with the dedupe Set redundant for the contiguous-rectangle path. The real cost is the High-tier history quadratic these changes feed.
- [ ] **`src/modules/grid/grid-editable-context.ts:77`** вЂ” the cell-slice selector allocates a `cellKey` string per cell per notification; window-bounded GC tax only. **Fix:** `extraCells.size > 0 && extraCells.has(...)`.
- [ ] **`src/modules/grid/grid-export.ts:48-58`** вЂ” CSV export materializes the full sorted model and builds the entire string synchronously, blocking the tab on tens-of-thousands-of-row exports. **Fix:** chunk/yield across frames or use a Web Worker; assemble the Blob from chunks.
- [ ] **`src/modules/grid/grid-table-options.ts:51-61`** вЂ” `toRowSelectionState` rebuilds an O(selected) object per selection change feeding per-render `buildState`; `state.rowSelection` is effectively write-only (no `getSelectedRowModel`/`getIsSelected` call sites), so there is no eager engine recompute. **Fix:** maintain incrementally.
- [ ] **`src/modules/grid/use-grid-navigation.ts:168-172`** вЂ” the navigation store notifies all listeners on every active-coord change (O(windowed-cells) cheap integer-compare selectors per keypress); a concern only for navigable + unvirtualized large grids. **Fix:** document that navigable requires `virtualize`, or index listeners by coordinate.
- [ ] **`src/modules/grid/grid-table-views.ts:155-193`** вЂ” `deriveVisibleColumns` re-resolves three engine leaf-column queries every render, unmemoized while sibling views are memoized; O(columns), tens of microseconds. **Fix:** key a `useMemo` on `columnOrder`/`columnVisibility`/`columnPinning` from `getState()`.
- [ ] **`src/modules/grid/grid-editable-cell.tsx:66-75`** вЂ” editable `aria-selected` is mirrored via `closest()` + `setAttribute` on mount for every windowed cell, writing `aria-selected="false"` universally and inflating the accessibility tree. **Fix:** emit only when true; apply reactively on the `<td>`.
- [ ] **`src/modules/grid/grid-row.tsx:398-405`** вЂ” `data-dragging` + an inline transform are toggled per windowed body cell of the dragged column during reorder; scoped to one column per pointer-move (others sit on NullContext), so bounded. Subsumed by the High-tier CSS-variable fix.
- [ ] **`src/modules/grid/use-grid-editable-history.ts:58`** вЂ” `priorValue`'s secondary `editableCols.find` per change is O(CВ·K), the lesser of the two scans in the High-tier quadratic; fixed by the same `columnIdв†’col` map.
- [ ] **`src/modules/grid/use-grid-editable-wrapper.ts:343-372`** вЂ” `collectMatrixChanges` allocates one `CellChange` per clipboard cell with no batching; intrinsic O(K) вЂ” the pathology is the High-tier inverse scan, not this allocation.
