# Documentation Audit — repo-wide

**Date:** 2026-06-22 · **Scope:** the **whole monorepo, comments included** — `packages/ui`, `packages/auth`, `packages/shared`, `apps/admin`, and the markdown surface docs. This widens the lens beyond the prior two passes ([2026-06-13](2026-06-13-DOC-AUDIT.md) surface, [2026-06-14](2026-06-14-DOC-AUDIT.md) ui comments-included): it re-verifies the `ui` surface against the ~180 source files that changed since 06-14, and runs the **first-ever** sweep over the non-`ui` workspaces (`auth`, `shared`, `admin`), which no prior audit covered. **Method:** the TSDoc standard (tags / syntax) was pulled from the documentation MCP knowledge base; the sweep was delegated one subsystem per subagent ([CLAUDE.md](../../../../CLAUDE.md) §3.3). **All edits are comment-only — no code, types, imports, JSX, or formatting changed**; a diff scan confirms zero non-comment changed lines. Verified: `biome check .` (1,613 files, no fixes) and `turbo run check-types` (tsc across all 4 packages incl. ui's scripts/docs configs) green.

This is a living record per [`CONVENTIONS.md`](../../../../CONVENTIONS.md) §12.3, accompanying the curated surface docs in [`../`](..).

## Executive summary

Since the 06-14 deep sweep the library churned heavily — the `Headless`→`HeadlessProvider` move under `providers/`, `Message`/`Text` `variant`→`severity`, severity-driven validation on Field/Control, DataTable column drag-reordering and the column-manager dialog, clearable DateInput/DatePicker, TagInput read-only-at-cap, NumberInput bare-variant buttons. The headline finding is that the public surface held: the 06-14 bar survived the refactors, and the renames were already reconciled in code comments and in the curated surface indices. The yield concentrated in two places — recipe-layer module headers that drifted as feature slots were added, and the never-audited non-`ui` workspaces, where the public surface simply had no TSDoc yet.

1. **`ui` public surface re-verified, no regressions.** The five named renames are clean everywhere: no comment still calls `Headless` a component, `severity` is consistently described, and `COMPONENTS.md` / `PROVIDERS.md` already match the barrels. Targeted gaps were filled (`SidebarLayout*`, `ControlPropsOptions`/`ControlPropsResult`, the two Kanban context hooks) and `@internal` summaries added on non-exported helpers across json-tree, stepper, tree, menu, recipe engine, typeahead.

2. **`auth` / `shared` / `admin` documented for the first time.** 22 files gained summary-first TSDoc on the public surface — `bifrost`, `getUser`/`User`, `withAuth`, `proxy`/`ProxyOptions`, `BIFROST_URL`; the chat hooks (`useChat`, `useSendMessage`) and types; the admin server fetchers and dialogs — plus `@internal` on the validator factories and form internals.

3. **Stale recipe headers and comments corrected.** The highest-value finds of a "comments included" sweep: kata module headers that no longer listed feature slots added by the refactor, and a kiso README file table that fell behind the wide-palette work.

4. **Standard normalization, continued.** Residual `@default`→`@defaultValue`, prose "Defaults to X" → `@defaultValue`, and bare symbol refs → `{@link}` in the changed files.

These are the comment-vs-code contradictions the sweep caught — recorded because they are the part most likely to mislead:

- **`components/tabs/tab-list-skeleton.tsx`** — said placeholders sit "above" the rail; they render as flex children inside the `k.list` row. Reworded.
- **`components/currency-input/currency-input.tsx`** — the `@remarks` editing-buffer claim had drifted from the current buffering behavior. Corrected.
- **`recipes/kata/tabs.ts`** — header sub-recipe list omitted the exported `scroll` overflow recipe. Added.
- **`recipes/kata/table.ts`** — `projection` doccomment referred to "the **old** tbody `striped` classes"; the `striped` projection key is current. Reworded to "below".
- **`recipes/kata/date-picker.ts`** — header slot list omitted the top-level `surface` and `control` slots (input-mode field adjacency). Added.
- **`recipes/kata/data-table.ts`** — chrome feature list omitted the `reorder` slot group (column drag-reorder). Added.
- **`recipes/kiso/README.md`** §3 — the `iro` row's Files column missed `ramp`, `project`, `spectrum` (added in the wide-palette work); Concern text now names the opt-in `spectrum` palette. Reconciled.

## Surfaced issues (code-level — left for follow-up)

Per [`CLAUDE.md`](../../../../CLAUDE.md) §1.2, flagged not fixed (a comment-only sweep can't touch code):

- **`components/input/input.tsx`** — `resolvedVariant` can fall back to `'glass'` (`variant ?? control?.variant ?? (glass ? 'glass' : undefined)`), but `InputProps.variant` is typed `'default' | 'outline'`. `'glass'` is a valid recipe variant yet not representable in the public prop type — a latent type/contract mismatch.
- **`shared/chat/use-send-message.ts`** — failure recovery drops the empty agent placeholder by content (`filter(m => !(m.role==='agent' && m.content===''))`), which would also remove any *other* empty agent message; the freshly-created `agentId` is the precise key. Harmless with one in-flight stream, but a refactor hazard.
- **`shared/chat/use-send-message.ts`** — each `content` SSE event *replaces* the agent bubble's content, so the gateway must stream cumulative snapshots; if it ever switches to deltas this silently truncates. A contract assumption worth a guard at the SSE boundary.
- **`recipes/kiso/kokkaku/control.ts`** — the header lists the `<ControlFrame>` consumer set (Input, Textarea, Select, Listbox, Combobox, DatePicker) but omits **color-picker-trigger**, which also consumes `ControlFrame`. Verifying the authoritative list is code-level.
- **`docs/demos/data-table.tsx`** — a block of commented-out dead code (old `BatchActionsExample` body) remains; candidate for deletion.
- **`pdf-viewer` "Fit to page" mislabel** and **`date-picker/date-picker-range-reducer.ts` byte-identical `pinEndpoint`/`hover` cases** — carried over unresolved from the [06-14 audit](2026-06-14-DOC-AUDIT.md); still open.
