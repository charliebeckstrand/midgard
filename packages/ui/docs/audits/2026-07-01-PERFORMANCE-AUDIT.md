# UI Grid Performance Audit — Open Findings

Performance audit of `src/modules/grid` (2026-07-01) — React 19, TanStack Table
v8 + Virtual v3, dnd-kit, motion. Method: a first-hand read of the render hot
path plus four parallel area audits (header/chrome, interaction hooks,
measurement/auto-size, data pipeline/bundle), every High/Medium finding verified
against the live source and the installed `table-core@8.21.3`. The architecture
holds: the row boundary is memoized end-to-end (`GridRow`/`GridDataCell`, engine
views element-wise ref-stabilized, cursor cells on an external store), editing
keystrokes stay in one editor, and closed overlays mount nothing. The real cost
now clusters in three places — a one-line sorting-state identity leak that
re-sorts the full row set on every render while any sort is active, N×M
full-grid invalidations at interaction boundaries (drag start/end, edit-mode
toggle, per-cell tooltip machinery under default `truncate`), and a statically
imported feature bundle (~630 KB min / 205 KB gzip) where ~35% serves features
that are off by default. Severity is priority order; line numbers are as of this
commit and will drift.

Since the [2026-06-28 audit](2026-06-28-PERFORMANCE-AUDIT.md): its top tiers are
largely resolved — `renderRows`/`rowKeys` now memoize on the engine's stable row
model (old H2), body cells no longer register per-cell sortables and glide on a
table-level CSS variable (old H6, old L12), `GridDataCell` is memoized (old M1),
selection flags are memoized behind a size guard (old M2), the selection `Set`
left `GridContext` and resize rides the narrow `GridResizingContext` (old M8),
search applies through local echo + `startTransition` (old H4), truncation
shares one document-level `ResizeObserver` (old M3), `deriveVisibleColumns` is
ref-stabilized (old L10), and `toRowSelectionState` memoized (old L8). The
editable-stack findings (old H1, H3, and five Lows) are moot — that subsystem
was rewritten onto the per-row model in #772, which this audit re-covers fresh
(H5 below). Still open and carried forward: the dep-less commit measure (old H5
residue, M7 below), the facet-button cost (old L2, upgraded to H6), the
comparator collator (old M4 refined into M2 — decorate-sort landed via
`sortKeyCache`, but see H1 for why it's currently defeated), CSV's main-thread
build (old L7), and column virtualization (backlog, `ROADMAP.md`).

## High

- [ ] **`src/modules/grid/use-grid-table.ts:375`** (with `grid-table-options.ts:127-148`) — an active sort re-sorts the entire row set on every GridData render. `buildState` is fed `sorting: toSortingState(sort)` inline — a fresh array each render — and table-core's row-model memo compares deps by identity (`getSortedRowModel.js:16`), so any render with `sorting.length > 0` recomputes the sorted model; worse, the engine's `sortData` spread-clones every row (`rows.map(row => ({...row}))`), so the per-`Row` WeakMap `sortKeyCache` — keyed on the clones — misses wholesale and every value re-parses through `toSortKey`. A 50k-row sorted grid pays O(n) clone + O(n) re-parse + O(n log n) compare + O(n) `renderRows`/`rowKeys`/`rowIndexMap` rebuilds per selection toggle, per committed search keystroke, per cursor move (M1), and per resize-drag frame — hundreds of ms per frame at scale. Row `original` identities and the spread-copied `getVisibleCells` closures survive, so the memoized DOM holds: this burns CPU, not reconciliation, which is why it hides. **Fix:** `useMemo(() => toSortingState(sort), [sort])` — `sort` identity from `useControllable` is already stable. Unsorted grids are unaffected (the engine short-circuits empty sorting to the same model reference).
- [ ] **`src/modules/grid/grid-data.tsx:298`** (with `use-grid-table.ts:188-209`, `grid-table-views.ts:203-213`, `use-grid-column-auto-size.ts:162,197-210`) — nothing stabilizes the consumer's `columns` prop at entry, so an inline array (the dominant React pattern) collapses the entire memo architecture: `resolveSortable` re-clones per render → new `columnDefs` + `meta.gridColumn` objects → TanStack rebuilds `Column` instances → per-row `getVisibleCells()` identity churns → every `GridRow`/`GridDataCell` memo is dead grid-wide, and the autosizer's `run` identity churn re-runs a full DOM measurement pass (`run(true)`, ~3–4 geometry reads per cell) on every render — selection toggles, keystrokes, cursor moves. `useStableColumnDefs`'s per-id renderer cache keeps cells from remounting, which masks the cost as CPU rather than lost focus. **Fix:** structurally stabilize `columns` at the boundary (extend the `sameElements` ref-hold with field comparison), and key the auto-size re-measure effect on the semantic signatures it already computes (`structSig`/`rowsSig`) through `runRef` instead of `run` identity.
- [ ] **`src/modules/grid/grid-cell-content.tsx:67`** (with `components/tooltip/use-tooltip-state.ts:60-127`, `hooks/use-media-query.ts:9-18`) — under default `truncate` every data cell permanently mounts a full floating-ui Tooltip stack: `useFloatingDisclosure` (controllable state, floating panel, dismiss, escape layer, role) + `useHover`/`useClick`/`useFocus`/`useInteractions`, a per-cell `matchMedia('(hover: hover)')` MQL + change listener — with `getSnapshot` constructing a *fresh* MQL on every cell render — and a dep-less effect running `matches(':disabled')` + `querySelector(':disabled')` after every cell commit. At 100×20 that is 2,000 stacks (~30k hook slots) paid at mount and re-paid continuously as virtualized rows churn through the window; closed tooltips attach no listeners and render no DOM, so all of it buys nothing at rest. **Fix:** one grid-level delegated tooltip (a single floating instance opened from the hovered truncated cell), or mount the stack lazily on first `truncated`/hover-intent; cache `useMediaQuery` MQLs module-level either way.
- [ ] **`src/modules/grid/grid-table-views.ts:227-243`** (with `grid-data.tsx:781`, `use-grid-truncation.ts:131,151-169`, `grid-row.tsx:320`) — a pointer drag-resize invalidates every data cell twice per gesture, at the two boundaries the freeze exists to protect: `useColumnSettleWidths` maps every entry to the `undefined` sentinel at drag start and back to numbers at settle, so all N×M `resizeSettleKey` props flip both times (every `GridDataCell` re-renders, re-runs its synchronous commit measure, and re-arms its rAF backstop — ~4 geometry reads per cell, twice), while the `GridResizingContext` boolean flip independently re-renders every `GridCellContent`. The "re-renders just that column's cells" contract (`grid-table-views.ts:222`) holds only for the keyboard nudge. Mid-drag frames hold as designed. **Fix:** freeze to the *previous* snapshot (`ref.current`) instead of `undefined` — drag start becomes a no-op and settle changes only the dragged column's entry — and skip the rAF backstop on transitions to `undefined`.
- [ ] **`src/modules/grid/grid-editing-context.ts:24`** (with `use-grid-editing.ts:189-192`, `grid-editing-cell.tsx:142`) — toggling one row into or out of edit mode re-renders every editing-aware cell in the grid: the memoized `rowEditing` context value necessarily changes identity with `editableRows`, and `createContext` has no selector, so all `GridEditingCell` consumers re-render through the `GridRow` memo — at 100×20, entering + saving one row is ~2×2,000 cell renders (each re-running the dep-less truncation measure, M7) to change 20 cells. Typing itself is clean (drafts stage in refs, one editor re-renders). **Fix:** thread a per-row `editing` boolean through `GridRow` props memo-scoped like `selected`, or subscribe cells by rowKey navStore-style.
- [ ] **`src/modules/grid/grid-column-filter-button.tsx:75-82`** (with `grid-table-views.ts:340-350`) — a *closed* select-type filter button materializes its facets during header render: `filter.uniqueValues(column.id)` builds TanStack's faceted unique-values Map over all rows on first paint, and the faceted model's memo deps include `globalFilter`/`columnFilters`, so every committed search keystroke re-pays O(rows) per select-filterable column — for a Sheet the user may never open (the Overlay unmounts it while closed). Even cache hits re-run the unmemoized `[...facets.keys()].filter().map()` → `new Set` → `.sort(localeCompare)` → `JSON.stringify` pipeline per header render — per drag-resize frame on a high-cardinality column. **Fix:** resolve facets on sheet open (or first `open`), cache the sorted list on the facet-Map reference, and key field identity off that reference instead of `JSON.stringify`.

## Medium

- [ ] **`src/modules/grid/use-grid-navigation.ts:178,391-398`** — the cursor coord is `useState` at the grid root solely to feed the `<table>`'s `aria-activedescendant`, so every arrow-key move re-renders the whole GridData tree — full hook pass (engine `setOptions`, colGroup rebuild, settle-width rescan), all header wrappers, N row memo compares — ~1–3 ms per keypress at 100×20, ~30×/s under key repeat, and each move re-triggers H1's re-sort while sorted. The store already scopes the actual cell flips to two cells. **Fix:** hold the coord in the store/ref only and write `aria-activedescendant` imperatively from the layout effect that notifies listeners (the module already does this for `data-active`).
- [ ] **`src/modules/grid/grid-sorting-utilities.ts:130`** (with `grid-table-views.ts:349`) — the sort-key string fallback runs `a.text.localeCompare(b.text, undefined, { numeric: true })` per comparison; engines fast-path only argument-less `localeCompare`, so each of the ~780k comparisons in a 50k-row sort resolves a collator — typically 5–20× slower than a hoisted one, multiplied by H1 while that stands. **Fix:** module-level `const COLLATOR = new Intl.Collator(undefined, { numeric: true })`; reuse it for the facet sort.
- [ ] **`src/modules/grid/use-grid-column-auto-size.ts:90-92,197-210`** — `rowsSig` (count + end row ids) changes on every committed client-filter keystroke, forcing `run(true)` — a full synchronous pre-paint DOM measurement pass (~3–4 geometry reads per rendered cell) per keystroke on the search path, inside the transition but still on the commit's critical path. The grow-only `runningContent` cache keeps the *write* from thrashing; the read pass is paid every time. **Fix:** defer rows-driven re-measures to rAF/idle (a one-frame-late fit is invisible), or debounce while `rowsSig` churns.
- [ ] **`src/modules/grid/use-grid-column-auto-size.ts:230-241`** (with `use-grid-truncation.ts:138`) — the font-settle re-measure is N-shot, not one-shot: keyed on `[enabled, run]`, it re-attaches `.then` to the already-resolved `document.fonts.ready` on every `run` identity change, clearing the grow-only cache and running a second full measure pass immediately after the one the same change already ran. The per-cell truncation hook has the sibling problem — each cell mount registers its own `fonts.ready.then(measure)`, doubling every row-mount's measures during virtualized scroll once fonts are loaded. **Fix:** subscribe once on mount through `runRef` (or gate on a settled flag); skip registration when `document.fonts.status === 'loaded'`.
- [ ] **`src/modules/grid/use-grid-column-auto-size.ts:216-228`** — the container `ResizeObserver` calls `runRef.current(false)` per tick with no coalescing or freeze, so a continuous window/panel drag writes fresh sizing per tick; each write lands element-wise changes in `settleWidths` → every row re-renders and every cell re-measures (commit + rAF backstop) per tick — dropped frames for the whole gesture on an unvirtualized grid. The RO tick itself reuses cached profiles; the cost is entirely downstream. **Fix:** trailing-debounce the sizing write (or extend H4's drag-freeze pattern to RO bursts).
- [ ] **`src/modules/grid/grid-head.tsx:714-732`** (with `grid-reorder.ts:145-153`) — during a column-reorder drag every `useSortable` header cell re-renders per pointermove (dnd-kit's ActiveDraggableContext bypasses the memo), and each re-render re-runs the header title's dep-less truncation measure (scrollWidth + `Range` + two rects — pure waste under a transform-only drag), the tooltip `:disabled` poll, and the matchMedia snapshot, while the dragged column's shift effect allocates and stops a fresh motion `animate()` per frame — ~20 columns × 60 fps ≈ 1,200 header renders + forced style reads per second. Body cells hold by design. **Fix:** gate the header measure while `isSorting` (mirror the `!resizing` gate) and drive the shift through one persistent motion value.
- [ ] **`src/modules/grid/use-grid-truncation.ts:131`** — carried from 2026-06-28: `useLayoutEffect(measure)` still has no dependency array, re-measuring every truncating cell on every commit — the multiplier that turns each full-pass render (H1, H2, H4, H5, M6) into an N×M text-geometry pass (`Range` + two rects per cell, the fitting-cell common case, ~10–30 ms at 2,000 cells). The observer side is now a shared singleton; the commit side remains unconditional. **Fix:** dep the commit measure on content-bearing inputs (`children`/settle key) and let the shared `ResizeObserver` own geometry-driven re-measures.

## Bundle

Measured with esbuild on `src/modules/grid/index.ts` (minified ESM, react/react-dom external): **~630 KB min / ~205 KB gzip**, all statically imported — no dynamic import anywhere in the module. Packaging itself is sound (`sideEffects: false`, per-module exports, no root barrel, lucide tree-shakes), so the wins are import-graph surgery:

- [ ] **`src/modules/grid/grid-reorder.ts:5-6`** — `motion` + `motion/react` retain ~134 KB min (≈21% of the whole bundle) for one CSS-variable spring and a `useReducedMotion` read, behind `reorder` which defaults **false**. **Fix:** `animate` from `motion/mini` (WAAPI, ~3 KB) or a hand-rolled spring; or lazy-load the reorder layer.
- [ ] **`src/modules/grid/grid-data.tsx:3-4`, `grid-head.tsx:3`** — @dnd-kit core+sortable+utilities (~48 KB min) ships in every grid despite `reorder=false` default (also reached by the dialog-gated group-manager editor). **Fix:** split the dnd wiring behind a lazy boundary keyed on `reorder`/manager-open.
- [ ] **`src/modules/grid/grid-body.tsx:5`** — `@tanstack/react-virtual` (~21 KB min) ships despite `virtualize` off by default; `GridVirtualizedBody` is already an isolated branch. **Fix:** `React.lazy` the virtualized body.
- [ ] **`src/modules/grid/grid-head.tsx` → `grid-column-filter-button.tsx` → `modules/query` barrel** — the QueryBuilder UI, `components/date-picker`, and `@internationalized/date` (~21 KB+ min combined) ship even when zero columns are filterable, and `grid-table-options.ts:23` imports the pure `evaluateQuery` through the same UI-carrying barrel. **Fix:** lazy the sheet body; give the evaluator a UI-free entry.

## Low

- [ ] **`src/modules/grid/grid-column-measure.ts:105-157,243`** — measurement iterates every rendered `<td>` with no sampling or read budget, and the content cap applies *after* reading everything; an unvirtualized, unpaginated 5k×10 grid pays ~150k geometry reads per pass. **Fix:** early-exit `bodyContentWidth` at the cap; sample K rows per column.
- [ ] **`src/modules/grid/grid-pinning.ts:22,48`** — `pinnedClassName`/`pinnedOffsetStyle` each call `column.getIsPinned()`, unmemoized in table-core 8.21.3 (allocates `getLeafColumns().map()` + two scans), twice per cell per body pass — GC noise at N×M. **Fix:** resolve `{side, offset}` once per visible column per render and pass the map down.
- [ ] **`src/modules/grid/grid-context-menu.tsx:315-318`** — `rowByKey` eagerly rebuilds an O(rows) Map on every row-model change while the menu is closed. **Fix:** resolve the row lazily at `contextmenu` time.
- [ ] **`src/modules/grid/grid-data.tsx:663`** (with `grid-group-head.tsx`) — the group band row resolves unmemoized every render and its cells (each with a Tooltip stack) are unmemoized, re-rendering per selection toggle/keystroke/drag frame. **Fix:** memo the header row on `[group, visibleColumns, pinning]`; memo `GridGroupHeadCell`.
- [ ] **`src/modules/grid/grid-group-manager.tsx:359-364`** (with `grid-reorder.ts:177-181`) — the group-name input commits per keystroke through `groupedColumnOrder`/`applyColumnReorder`, which returns a fresh array even when idempotent → engine column re-resolution + whole-grid render per typed character while the dialog is open. **Fix:** draft locally, commit on blur; return the input array when unchanged.
- [ ] **`src/modules/grid/grid-head.tsx:121-137`** — `GridHead` is unmemoized, so each resize-drag frame re-runs all header-cell wrapper bodies (memo'd children bail); linear in columns, noticeable past ~100. **Fix:** memo `GridHead` — its props are already identity-stable.
- [ ] **`src/modules/grid/grid-table-options.ts:77-89`** (with `use-grid-table.ts:218-222`, `ROADMAP.md:9`) — `usesClientModel` is true for every default grid (`clientSort` defaults true), so the documented plain-grid row-model bypass is dead code and the full `Row` set materializes on mount; `renderGridRow`'s `table.getRow` would force it regardless. **Fix:** correct the docs, or gate `getSortedRowModel` wiring on `sortable && sort.length > 0`.
- [ ] **`src/modules/grid/use-grid-navigation-columns.tsx:37-62,105`** — each cursor activation walks every header cell with `getComputedStyle` + `getBoundingClientRect` for `obscuringInsets` (~20 reads per keypress, ~600/s under repeat); deliberate for WCAG 2.4.11 but cacheable. **Fix:** epoch-key the inset cache on settle-widths/pin/density identity.
- [ ] **`src/modules/query/query-builder/query-builder-evaluate.ts:40-42,101`** — per-row filter evaluation allocates a rest-spread + closure per row and lowercases the *rule* value per row; a few ms GC churn per 100k-row pass. **Fix:** index loop; hoist the lowered needle.
- [ ] **`src/hooks/use-media-query.ts:9-18`** — `getSnapshot` constructs a fresh `MediaQueryList` on every subscriber render and `subscribe` builds one per subscriber; N×M via the per-cell tooltip (H3). **Fix:** module-level query→MQL cache with a shared listener.
- [ ] **`src/components/table/table-cell.tsx:20`** — every cell render runs the recipe (`clsx`+`twMerge`) plus `cn` (`clsx`+`twMerge` again) for a no-arg `k.cell()` whose result is constant; tailwind-merge's LRU absorbs most of it after warmup, but it sits in the N×M hot loop. **Fix:** hoist the no-arg recipe result to a module constant.
- [ ] **`src/modules/grid/use-grid-column-auto-size.ts:95`** — `structSig` joins ids with `''`, so `["ab","c"]` and `["a","bc"]` collide and suppress the cache clear — perf-adjacent correctness. **Fix:** join with `' '`.
- [ ] **`src/modules/grid/use-grid-table.ts:186-199`** — `useStableColumnDefs`'s renderer Map never evicts ids no longer in `columns`; unbounded only under dynamically generated column ids. **Fix:** prune on rebuild.
- [ ] **`src/modules/grid/export/export-csv.ts`** — carried from 2026-06-28: the full CSV string still builds synchronously on the main thread; tens-of-thousands-of-row exports block the tab. **Fix:** chunk across frames or a worker; assemble the Blob from chunks.
